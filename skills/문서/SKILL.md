# 일관 스타일 HTML 문서 생성

## 발동 조건 (중요)

이 스킬은 **명시적 트리거가 있을 때만** 동작한다 — `/doc-style:문서 ...` 슬래시 호출(또는 이 PC 환경의 `!문서 ...`).
접두사 없이 "회의록 만들어줘", "매뉴얼 써줘", "문서로 정리해줘" 같은 일반 요청에는 **발동하지 말 것**. 사용자가 의도치 않게 이 스타일이 튀어나오는 것을 원치 않는다. 명시 트리거가 없으면 평범하게 대응한다.

---

명시적으로 호출되면, 요청한 문서를 **고정된 디자인 시스템**으로 자기완결 HTML 한 파일로 만든다.
스타일을 새로 디자인하지 말 것 — 아래 자산을 그대로 사용하는 것이 이 스킬의 존재 이유다.

## 자산 (반드시 사용)

- 디자인 시스템 CSS: `${CLAUDE_PLUGIN_ROOT}/skills/문서/doc-style.css` ← 스타일의 단일 출처(SSOT)
- 구조 템플릿:       `${CLAUDE_PLUGIN_ROOT}/skills/문서/template.html` ← 블록 마크업 패턴 참조

## 절차

1. `doc-style.css`와 `template.html`을 읽는다.
2. 사용자가 요청한 내용으로 **본문만** 채운다. 문서 종류에 맞는 블록을 골라 조합한다(아래 "블록" 참조). 불필요한 블록은 뺀다.
3. 최종 산출물은 **자기완결 단일 HTML**로 만든다 — `doc-style.css`의 내용을 통째로 `<head>` 안의 `<style>`에 인라인한다(외부 링크·CDN 금지, 어디로 보내도 깨지지 않게).
4. 저장 위치: 사용자가 경로를 지정하면 거기에, 아니면 현재 작업 폴더에 `<문서명>.html`로 저장한다.
5. 저장 후 브라우저로 열어 보여준다(Windows: `Start-Process <경로>`).

## 블록 (template.html의 마크업을 그대로 복제해 내용만 교체)

- **헤더** `header.doc` — eyebrow(분류) + h1(제목) + subtitle(부제) + meta 그리드(일시·작성자·버전 등 2열). 그라데이션 강조.
- **요약** `section.card.summary` — 좌측 강조 바. 문서 전체를 1~3문장 요약.
- **번호 섹션** `section.card` + `h2 > span.num` — 본문/리스트/소제목(h3)/코드(pre)를 담는 기본 카드. num에는 1,2,3… 또는 ·, →, ✓ 같은 기호.
- **콜아웃** `.callout.info`(참고) / `.callout.warn`(주의) — 강조 박스.
- **표** `.table-wrap > table.spec` — `th`는 헤더, `td.key`는 키 강조, `.tag.req`(필수)·`.tag.opt`(선택) 알약. table-wrap이 모바일 가로 스크롤을 보장하므로 표는 항상 table-wrap으로 감싼다.
- **체크리스트/결정사항** `section.card.checklist` — 초록 배경 + ✓ 마커. 요점·결정·점검 목록.
- **푸터** `footer.doc` — 출처·링크·주석.

## 불변식 (절대 깨지 않는다)

- `:root` 토큰(색·여백·radius)과 폰트 스택은 그대로 둔다.
- `word-break:keep-all` (한글 줄바꿈), `clamp()` 반응형 폰트 유지.
- `@media print`(단락 `break-inside:avoid`, 그라데이션 `print-color-adjust:exact`)와 `@media screen (max-width:640px)`(모바일) 블록을 **그대로 포함**한다. A4 인쇄 시 단락이 페이지 경계에서 잘리지 않아야 하고, 모바일에서 반응형으로 변해야 한다.
- 인쇄/PDF 버튼(`.toolbar.no-print`)을 유지한다.

## 색만 바꾸고 싶을 때 (요청이 있을 때만)

- 포인트 색: `:root`의 `--accent`, `--head-from`, `--head-to`만 교체. (예: 더 밝은 파랑이면 `--head-from:#0b5f9e; --head-to:#14406e;`)
- 구조·레이아웃·폰트는 바꾸지 않는다.

## 금지

- **산출물은 HTML 단일 파일이다.** docx(Word)·PDF 등 다른 형식으로 변환하거나 추가로 생성하지 말 것. 워드·PDF가 필요하면 사용자가 그 형식을 명시적으로 요청했을 때만 별도로 처리한다. (회의록 등을 만들 때 docx 스킬을 곁들여 발동하지 말 것)
- 디자인을 새로 만들거나 다른 테마(파스텔/다크/터미널 등)로 바꾸지 말 것. **일관성이 목적이다.**
- 폰트·여백·카드 구조를 임의로 변경하지 말 것.
- 외부 스타일시트·웹폰트 CDN에 의존하지 말 것(자기완결 원칙).
