핵심 자산(디자인 CSS·스킬)을 GitHub에 올려둬서, 다른 컴퓨터에선 처음부터 다시 만들 필요 없이 가져다 쓰기만 하면 됩니다. 같은 계정으로 Cowork를 쓰면 그것조차 자동이고, 그 외에는 아래 한 번의 설정으로 끝납니다.

1환경별 설정 방법

다른 컴퓨터에서 어디서 쓰느냐에 따라 셋 중 하나입니다.

환경해야 할 일
Cowork · 같은 claude.ai 계정아무것도 안 함 자동 — 개인 스킬이 계정에 저장돼 로그인만 하면 그대로 있음
Cowork · 다른 계정스킬 한 번 재등록 2번 참고
Claude Code (터미널·데스크톱)plugin 설치 한 번 3번 참고
가장 흔한 경우

같은 계정으로 Cowork를 쓰면 그냥 자동입니다. 로그인 후 바로 !문서를 치면 됩니다.

2Cowork에 스킬 재등록 (다른 계정일 때)

Cowork 왼쪽 사용자 지정(Customize) → 스킬 → +(스킬 지침 작성)을 열고, 아래 세 칸을 채운 뒤 만들기를 누릅니다.

① 스킬 이름

doc-style

② 설명

회의록·매뉴얼·리포트 등을 사내 doc-style(네이비 헤더·2열 메타·번호 섹션 카드)의 자기완결 HTML로 만든다. 문서를 이 스타일로 만들 때 사용.

③ 지침

회의록·매뉴얼·리포트 등 문서를 만들 때는 항상 아래 CSS를 fetch해서 <style>에 그대로 인라인하고, 그 디자인으로만 만든다. 기본/임의 스타일 금지.
- 헤더: 네이비 그라데이션(#2b4c8c → #16294d) + 흰 글씨
- 메타 정보: .meta 2열 텍스트 (pill·아이콘·네비게이션 바 금지)
- 색·레이아웃을 새로 발명하지 말 것
- 결과는 외부 의존 없는 자기완결 HTML 단일 파일
CSS: https://raw.githubusercontent.com/fkrn75/doc-style-plugin/main/skills/%EB%AC%B8%EC%84%9C/doc-style.css
참고

등록 후 Cowork에서 !문서를 치면 '문서'라는 단어가 설명과 매칭돼 자동 발동하고, /doc-style로 명시 호출도 됩니다.

3Claude Code(터미널·데스크톱)에 plugin 설치

대화형 claude의 입력창에서 아래를 차례로 실행합니다.

/plugin marketplace add fkrn75/doc-style-plugin
/plugin install doc-style@doc-style

설치 후 /doc-style:문서로 호출하면 됩니다. 디자인 CSS도 함께 따라오므로 색·모양이 정확합니다.

주의

Cowork에는 /plugin 명령이 없습니다. 이 방법은 터미널/데스크톱 앱의 Claude Code 전용입니다.

4왜 이렇게 갈리나 (핵심 원리)

  • 로컬 Claude Code는 PC의 파일(CLAUDE.md·doc-style.css)을 직접 읽으므로 !문서가 그대로 동작합니다.
  • Cowork는 로컬 파일을 일절 못 읽고 외부 URL만 읽습니다. 그래서 CSS를 GitHub raw URL로 두고, 스킬이 그 URL을 fetch하게 한 것이 핵심입니다.
  • 그래서 모든 환경이 같은 GitHub 저장소 하나를 바라보게 됩니다 — 디자인을 한 곳만 고치면 전부 반영됩니다.
